# Security Policy

## Reporting Vulnerabilities

Report security vulnerabilities via GitHub Security Advisories on the relevant repository. Do NOT open public issues for security concerns.

All reports will be reviewed promptly. We take every report seriously.

## Security Principles

- **Privacy is fundamental.** No telemetry, no analytics, no tracking.
- **Independence.** Our projects run standalone. They do not phone home.
- **No secrets in source.** Credentials, keys, and connection strings live in environment variables or local configuration, never in committed code.
- **Equal respect.** The same security standards apply to all participants -- human or synthetic.

## Dependency Management

- Dependencies are tracked and updated via Renovate and Dependabot.
- Security scanning is enabled via CodeQL, Qodana, and Codacy where applicable.

## What We Do NOT Guarantee

- No SLA on vulnerability response time (small team, real life).
- No formal security audit has been performed.
- No warranty of any kind. See each project's LICENSE.
